# ⚙️ Track 04 — Backend (FastAPI + Databases) 🔒 after Python

Unlock this once you're comfortable with Python (CS50P, [Track 01](../01-python)). Backend is how your apps get a server, store data, and talk to other software — and later, how AI models get **served** to real users.

## The path (when you get here)
1. **See the whole map** — skim the backend roadmap → <https://roadmap.sh/backend>
2. **APIs with FastAPI** — modern, fast, and what you'll actually use. The official tutorial is excellent → <https://fastapi.tiangolo.com>
   - *Alternative, famously step-by-step:* Miguel Grinberg's **Flask Mega-Tutorial**.
3. **Databases** — SQL basics, then connect a database to your API (SQLite → Postgres).
4. **Build something real** — a REST API (a notes or tasks app), with a database behind it. Then deploy it.

## Checklist (preview)
- [ ] Understand the flow: client ↔ server ↔ database
- [ ] Build your first API endpoint with FastAPI
- [ ] Add a database; do create / read / update / delete (CRUD)
- [ ] Ship a small full app — frontend from [Track 02](../02-web) talking to this API

> This track pairs with the web track to make you a **full-stack builder** — and it sets up AI engineering in [Track 05](../05-ai-ml), where models get served through exactly this kind of backend.
