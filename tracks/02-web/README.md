# 🌐 Track 02 — Web Fundamentals (HTML · CSS · JS)

Goal: **build things people can see.** This is where "I made something real" happens fastest — and it's how your portfolio gets built.

## The path
1. **HTML + CSS** — structure and style.
2. **JavaScript** — make it interactive.
3. **Build, build, build** — small pages first, then your portfolio.

## Pick one main resource (don't tab-hop)
- **The Odin Project — Foundations** — the best structured, project-based path → <https://www.theodinproject.com>
- **MDN — Getting started with the web** — gentle intro + reference → <https://developer.mozilla.org/en-US/docs/Learn>
- **freeCodeCamp — Responsive Web Design** — interactive, in-browser → <https://www.freecodecamp.org/learn>
- For CSS that finally *clicks*: **Josh Comeau's articles** → <https://www.joshwcomeau.com>

## Checklist
- [ ] HTML basics — tags, structure, links, images
- [ ] CSS basics — the box model, flexbox, colors, fonts
- [ ] Build a static **profile card** page
- [ ] JavaScript basics — variables, functions, the DOM, events
- [ ] Add interactivity to the card (a toggle, a counter — anything)
- [ ] **Portfolio v1** — a clean homepage about you, **deployed live**
- [ ] Make it **responsive** — looks right on a phone
- [ ] **Portfolio v2** (later) — add a dynamic piece: a blog, or a project showcase

## Your project
Your portfolio lives in `projects/portfolio/`. Start **static (v1)** to nail the build → deploy → Git loop, then **grow it (v2+)** over the year. It doubles as living proof you can build.

> **Deploy free:** **GitHub Pages** (perfect for static sites) or **Vercel** (great once you add JS frameworks). Getting it live on the internet is the whole point — localhost doesn't count.
